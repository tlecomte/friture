## Context

Feature/global-calibration has landed. The calibration infrastructure is in place:
GlobalCalibrationService, MicCalFile, per-dock overrides, spectrum/octave/spectrogram
frequency correction. This sprint closes the remaining calibration gaps and adds the
workflow touches that bring the FOH engineer experience closer to the "under 2 minutes
to calibrated data" goal from the product vision.

## Vision

### 10x Check

Today's calibration flow: open Settings → enter a dB number → optionally browse for
a .cal file → close Settings. Fast, but opaque.

A 10x calibration experience:
- The user plugs in their measurement mic and connects their calibrator
- A live SPL meter appears in the dB Levels dock showing the current reading
- They hold the calibrator to the mic and press "Calibrate" — the number locks in
- A named profile is saved: "Focusrite Solo @ Venue X, 2026-06-13"
- Switching devices: the app detects the known device and loads its saved profile
- "Last calibrated: 3 days ago with [device]" visible at all times

The gap between today and 10x is: opacity (no feedback, no memory, no history).
This sprint closes the memory gap. The live-meter wizard is the next sprint.

## Architecture Decisions (resolved during CEO review)

**Key schema (cross-model tension resolved):**
`GlobalCalibration/profiles/{device_key}/offsetDb`
- `device_key` = sanitized composite of `(name, host_api_name)` from PortAudio
- Profiles-compatible for future named profiles (no migration needed when profiles land)
- Legacy flat keys (`GlobalCalibration/offsetDb`) kept as read-only fallback for 2+ releases
- Never delete old keys during upgrade (prevents calibration loss on downgrade)

**Device identity source (cross-model tension resolved):**
Read from live `InputDeviceCatalog` (the real active audio device), not from QSettings.
Passes the device identity as a param into `saveState(settings, device_key)`.

**Calibrate logic (S1-A resolved):**
Extract `calibrate_from_live_reading()` into `GlobalCalibrationService`.
Both Settings_Dialog and dB Levels dock button call this method directly.
No circular import.

**Stale mic cal warning (S4-A resolved):**
Check `os.path.exists(mic_cal_file_path)` in `_sync_mic_cal_form`, not `mic_cal is None`.
Covers both missing-file and corrupt-file cases.

**Quick-recalibrate UX (S11-A resolved):**
Toolbar icon in dB Levels dock → calibrate_from_live_reading() → status bar message.
No modal, no Settings required.
